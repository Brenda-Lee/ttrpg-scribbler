# Task 13: Sheet primitive field components

## Overview
Build the six "primitive" sheet field components — `TextField`, `TextareaField`, `NumberField`, `CheckboxField`, `SelectField`, `DerivedField` — under `src/components/characters/sheet/fields/`. Each wraps the appropriate ShadCN primitive with React Hook Form's `Controller`, exposes a `name`/`control`/`field` prop contract, and surfaces breakdown tooltips for derived fields.

<requirements>
- MUST create six components under `src/components/characters/sheet/fields/`: `TextField.tsx`, `TextareaField.tsx`, `NumberField.tsx`, `CheckboxField.tsx`, `SelectField.tsx`, `DerivedField.tsx`.
- MUST accept props of shape `{ name: string; control: Control<SheetFormValues>; field: SheetField; breakdown?: BreakdownEntry[] }` and use RHF's `Controller` to bind input changes.
- MUST render `label` from `field.label` when present, falling back to `field.id`.
- MUST display the `field.unit` suffix (e.g., "m" for Deslocamento) when present.
- MUST `NumberField` accept text input and `valueAsNumber: true` (or equivalent) so RHF stores numbers, not strings.
- MUST `SelectField` render only options declared in `field.options`.
- MUST `DerivedField` be read-only and show a Tooltip listing `breakdown[]` entries with `+/-delta · reason` lines.
- MUST NOT include `RepeatingListField` (task 14 owns that component).
- SHOULD share a small `<FieldShell label unit>` wrapper for the label/unit layout if it reduces duplication.
</requirements>

## Implementation Details
See TechSpec section "System Architecture → Client-side" and ADR-003 for the RHF integration. ShadCN primitives `input`, `textarea`, `tooltip`, `badge`, and `label` already exist; `select` and `checkbox` are generated by task 01. Field components are intentionally thin — orchestration logic (defaults, autosave, derivation) lives in the renderer (task 15).
